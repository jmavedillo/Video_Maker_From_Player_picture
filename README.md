# Video_Maker_From_Player_picture

Minimal isolated experiment service that generates a short MP4 from:
- one provided base image (or optional local fallback)
- one long scrolling text line (music-player style reveal)

Implementation is intentionally simple:
- plain Node.js + built-in HTTP server
- direct `ffmpeg` command (no heavy wrapper libs)
- Docker-first runtime for Railway

## Project structure

```txt
src/
  generator.js      # reusable ffmpeg generator logic
  server.js         # HTTP service
  index.js          # one-shot local generator entrypoint
assets/
  images/
    base.png        # required (you provide)
  fonts/
    *.ttf|*.otf     # required (you provide one font)
output/
  test-reveal.mp4   # generated by npm run generate
```

## Required assets

This repo does **not** include production assets.
You must add:

1. `assets/images/base.png`
   - any PNG image (recommended landscape, e.g. 1280x720)
2. One font file in `assets/fonts/`
   - `.ttf` or `.otf` (example: `DejaVuSans.ttf`)

Missing assets fail with explicit error messages.

## API

### `GET /health`
Returns 200 with JSON:

```json
{"ok":true}
```

### `POST /render`
Accepts JSON body:

```json
{
  "text": "your reveal text",
  "baseImage": {
    "dataBase64": "<base64 PNG>"
  },
  "videoDurationSeconds": 10,
  "scrollStartSeconds": 1.5,
  "scrollEndSeconds": 8.5
}
```

- `text` is required.
- `baseImage.dataBase64` is the caller-provided PNG bytes encoded as base64.
- if `baseImage` is omitted, the service falls back to local `assets/images/base.png`.
- timing fields are optional and default to environment variables or built-ins.
- response is `video/mp4`.

Validation errors include:
- missing/empty `text`
- invalid `baseImage` object shape
- missing/empty `baseImage.dataBase64`
- invalid base64
- invalid or unreadable decoded image

Timing values must satisfy:
- `VIDEO_DURATION_SECONDS > 0`
- `0 <= SCROLL_START_SECONDS < SCROLL_END_SECONDS <= VIDEO_DURATION_SECONDS`

## Local run

Start service:

```bash
npm start
```

One-shot file generation (old behavior):

```bash
npm run generate
```

Example render request with JSON base64 image input:

```bash
BASE64_IMAGE="$(base64 -w 0 assets/images/base.png)"

curl -X POST http://localhost:3000/render \
  -H "Content-Type: application/json" \
  --data-binary "$(jq -n \
    --arg text 'Now Playing — Your long custom title goes here' \
    --arg base64 "$BASE64_IMAGE" \
    '{
      text: $text,
      baseImage: { dataBase64: $base64 },
      videoDurationSeconds: 12,
      scrollStartSeconds: 2,
      scrollEndSeconds: 10
    }')" \
  --output render.mp4
```

## Docker

Build image:

```bash
docker build -t video-maker-poc .
```

Run container:

```bash
docker run --rm -p 3000:3000 video-maker-poc
```

## Railway deployment (from GitHub)

1. Push this repo to GitHub.
2. In Railway, create a new project and choose **Deploy from GitHub repo**.
3. Select this repository.
4. In service settings, ensure Railway uses the **root `Dockerfile`**.
5. Add required assets into the repo (`assets/images/base.png` and a font in `assets/fonts/`).
6. Deploy.

### Railway runtime notes

- Service listens on `process.env.PORT`.
- Healthcheck path should be set to:
  - `/health`
- Generate a public URL/domain:
  - Open your Railway service → **Settings** → **Networking** → **Generate Domain**.

### Optional environment variables

- `REVEAL_TEXT`
- `VIDEO_DURATION_SECONDS`
- `SCROLL_START_SECONDS`
- `SCROLL_END_SECONDS`

These values are used as defaults when request timing fields are omitted.

## Scope

This stays an isolated experimental service.
No frontend, auth, database, queue, or persistent storage is added.
